# Memory Keeper

解决 OpenClaw AI `/new` 后"失忆"问题的完整记忆管理方案。核心目标：**工作不因 new 而中断**。

---

## 安装后配置（必做一次）

将以下内容追加到 `~/.openclaw/workspace/AGENTS.md`：

```markdown
## 记忆管理（memory-keeper）

### Session 启动时执行（三层加载）

1. **热层**：读取 `memory/tasks.md`（进行中的任务）
   - 有"进行中"的任务 → 主动告知用户："上次在做 [任务名]，下一步是 [具体操作]，继续吗？"
   - 无进行中任务 → 跳过

2. **温层**：读取 `memory/YYYY-MM-DD.md`（今天 + 昨天的日记）
   - 不存在今天的日记 → 用日记模板创建
   - 存在 → 读取恢复上下文

3. **冷层**：仅在涉及具体项目时才读取 `MEMORY.md`（项目索引）

### 工作状态记录时机（必须执行，不等用户 /new）

遇到以下情况时，立即更新 `memory/tasks.md`：
- 完成一个明确的里程碑（发布版本、解决 bug、完成功能模块）
- 用户说"好，先这样"、"今天到这里"、"暂停"等暗示阶段结束的话
- Context 使用率 > 80% 时主动保存当前状态并提醒用户

### 项目变更时执行
当发生以下任何操作时，读取 `skills/memory-keeper/SKILL.md` 并按其规范更新 `MEMORY.md` 的项目索引：
- 新建/删除项目或 skill
- 发布新版本
- 更改 Git 地址或本地目录
```

> 💡 这些规则是让 AI 主动执行记忆管理的关键。不写进 AGENTS.md，AI 只有在被触发时才会读 SKILL.md，无法自动执行。安装 skill 后必须完成此配置步骤。

---

## Part 1：工作状态管理（核心）

### tasks.md 格式

```markdown
# 任务状态

## 进行中
- [ ] {任务名称}
  - 当前状态：{一句话说明做到哪里了}
  - 下一步：{具体可以直接执行的操作，足够清楚让新 session 无需追问就能开始}
  - 记录时间：{YYYY-MM-DD HH:MM}

## 已完成（存档）
- [x] {任务名称}（完成：{YYYY-MM-DD}）
```

### 何时更新 tasks.md

| 触发条件 | 操作 |
|---------|------|
| 完成里程碑 | 把任务从"进行中"移到"已完成"，并记录完成日期 |
| 开始新任务 | 新建"进行中"条目，填写当前状态和下一步 |
| 任务状态变化 | 更新"当前状态"和"下一步"字段 |
| 用户暗示暂停 | 确保"下一步"字段已填写完整再记录 |
| Context > 80% | 主动保存当前状态，告知用户建议 `/new` |

### 启动时的恢复话术

发现有进行中的任务时，主动说：

> 上次我们在做「{任务名}」，做到了「{当前状态}」，下一步是「{下一步}」。要继续吗？

用户说"是" → 直接执行下一步  
用户说"否" → 跳过，等待用户新指令

### 已完成任务的管理

- 已完成任务标记为 `[x]` 保留在"已完成"区域
- 用户可以随时手动删除不需要的历史记录
- 超过 30 天的已完成任务可以提示用户是否归档清理

## Part 2：每日日记

### 日记模板

```markdown
# YYYY-MM-DD

## 今日工作
- 

## 重要决策
- 

## 遇到的问题
- 问题：
  解决方案：

## 经验教训
- 

## 待办
- [ ] 
```

### 必须记录的情况

| 触发 | 记录内容 |
|------|----------|
| 用户做出决策 | 决策内容 + 理由 |
| 解决了一个问题 | 问题描述 + 解决方案 |
| 用户说"记住这个" | 完整内容 |
| 重要配置变更 | 变更前后 + 原因 |
| session 出现错误 | 错误内容 + 教训 |

### Session 过大时的主动处理

**优先用 context 使用率**（通过 `session_status` 工具获取）：

| Context 使用率 | 状态 | 我的行动 |
|--------------|------|----------|
| < 60% | ✅ 健康 | 正常使用 |
| 60–80% | ⚠️ 注意 | 确保日记和任务状态已写入 |
| > 80% | 🔴 主动处理 | 保存工作状态，告知用户建议 `/new` |

## Part 3：项目索引

### 存储位置

项目索引维护在 `MEMORY.md` 的 `## 项目索引` 章节中。

### 项目条目格式

```markdown
### {项目名称}
- **用途**：{一句话说明这个项目是什么}
- **本地目录**：`{绝对路径}`
- **Git 地址**：{https://github.com/... 或 "无（本地项目）"}
- **当前版本**：{版本号或状态描述}
- **最后更新**：{YYYY-MM-DD}
- **注意**：{关键注意事项，下次操作前必须知道的事}
```

### 必须更新项目索引的时机

| 触发 | 操作 |
|------|------|
| 新增项目/skill | 新建条目 |
| 发布新版本到 ClaWHub/GitHub | 更新版本号和最后更新日期 |
| 更改 Git remote 或本地目录 | 更新对应字段 |
| 删除项目/skill | 从索引中移除对应条目 |
| 发现重要注意事项 | 追加到"注意"字段 |

> ⚠️ **任何项目操作完成后，必须同步更新 MEMORY.md 项目索引。** 创建、删除、版本更新都算——操作完不更新索引等于没记录。

---

## 注意事项

- 日记和 tasks.md 不要记录密钥、token 等敏感信息
- tasks.md 的"下一步"要写得足够具体，让全新 session 的 AI 无需追问就能直接执行
- 项目索引放在 MEMORY.md 最前面，方便新 session 快速扫描
