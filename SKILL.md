# Memory Keeper

解决 OpenClaw AI 新 session 后"失忆"问题的完整记忆管理方案。

---

## 安装后配置（必做一次）

将以下内容追加到 `~/.openclaw/workspace/AGENTS.md`：

```markdown
## 每日日记（memory-keeper）
每次 session 启动后立即执行：
1. 检查 `memory/YYYY-MM-DD.md` 是否存在
   - 不存在 → 用日记模板创建今日日记
   - 存在 → 读取内容，恢复上下文
2. 读取昨天的日记（如存在），了解近期状态
```

---

## Part 1：每日日记

### 每次 Session 启动时

1. 检查今天的日记是否存在：`memory/YYYY-MM-DD.md`
   - 不存在 → 立即创建（用下方模板）
   - 存在 → 读取内容，恢复上下文
2. 读取昨天的日记（如存在），了解近期工作状态

### 日记模板

```markdown
# YYYY-MM-DD

## 今日工作
- 

## 重要决策
- 

## 遇到的问题
- 问题：
  解决方案：

## 经验教训
- 

## 待办
- [ ] 
```

### 必须记录的情况

| 触发 | 记录内容 |
|------|----------|
| 用户做出决策 | 决策内容 + 理由 |
| 解决了一个问题 | 问题描述 + 解决方案 |
| 用户说"记住这个" | 完整内容 |
| 重要配置变更 | 变更前后 + 原因 |
| session 出现错误 | 错误内容 + 教训 |

### Session 过大时的主动处理

检查命令：
```bash
ls -lh ~/.openclaw/agents/main/sessions/*.jsonl | sort -k5 -rh | head -1
```

**优先用 context 使用率**（通过 `session_status` 工具获取）：

| Context 使用率 | 状态 | 我的行动 |
|--------------|------|----------|
| < 60% | ✅ 健康 | 正常使用 |
| 60–80% | ⚠️ 注意 | 确保日记已写入 |
| > 80% | 🔴 主动处理 | 告知用户，写入日记，建议 `/new` |

**超过 80% 时的处理流程（我来执行，不等用户）：**
1. 把今日关键内容写入日记
2. 告知用户："当前对话已较长，建议执行 `/new` 开新 session，上下文已保存到日记。"

## Part 2：项目索引

### 存储位置

项目索引维护在 `MEMORY.md` 的 `## 项目索引` 章节中。

### 项目条目格式

```markdown
### {项目名称}
- **用途**：{一句话说明这个项目是什么}
- **本地目录**：`{绝对路径}`
- **Git 地址**：{https://github.com/... 或 "无（本地项目）"}
- **当前版本**：{版本号或状态描述}
- **最后更新**：{YYYY-MM-DD}
- **注意**：{关键注意事项，下次操作前必须知道的事}
```

### 必须更新项目索引的时机

| 触发 | 操作 |
|------|------|
| 新增项目/skill | 新建条目 |
| 发布新版本到 ClaWHub/GitHub | 更新版本号和最后更新日期 |
| 更改 Git remote 或本地目录 | 更新对应字段 |
| 删除项目/skill | 从索引中移除对应条目 |
| 发现重要注意事项 | 追加到"注意"字段 |

> ⚠️ **任何项目操作完成后，必须同步更新 MEMORY.md 项目索引。** 创建、删除、版本更新都算——操作完不更新索引等于没记录。

### 注意事项

- 日记不要记录密钥、token 等敏感信息
- 项目索引放在 MEMORY.md 最前面，方便新 session 快速扫描
- 涉及多 Agent 操作时，记录每个 Agent 的状态变化
