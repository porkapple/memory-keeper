# Memory Keeper

解决 OpenClaw AI 新 session 后"失忆"问题的完整记忆管理方案。

---

## 安装后配置（必做一次）

将以下内容追加到 `~/.openclaw/workspace/AGENTS.md`：

```markdown
## 记忆管理（memory-keeper）

### Session 启动时执行
每次 session 启动后立即执行：
1. 检查 `memory/YYYY-MM-DD.md` 是否存在
   - 不存在 → 用日记模板创建今日日记
   - 存在 → 读取内容，恢复上下文
2. 读取昨天的日记（如存在），了解近期状态
3. 读取 `memory/tasks.md`（如存在），检查是否有未完成任务
   - 有未完成任务 → 主动告知用户："上次有未完成的工作：[任务列表]，是否继续？"

### 项目变更时执行
当发生以下任何操作时，读取 `skills/memory-keeper/SKILL.md` 并按其规范更新 `MEMORY.md` 的项目索引：
- 新建/删除项目或 skill
- 发布新版本
- 更改 Git 地址或本地目录
```

> 💡 这三条规则是让 AI 主动执行记忆管理的关键。不写进 AGENTS.md，AI 只有在被触发时才会读 SKILL.md，无法自动执行。安装 skill 后必须完成此配置步骤。

## Part 1：每日日记

### 日记模板

```markdown
# YYYY-MM-DD

## 今日工作
- 

## 重要决策
- 

## 遇到的问题
- 问题：
  解决方案：

## 经验教训
- 

## 待办
- [ ] 
```

### 必须记录的情况

| 触发 | 记录内容 |
|------|----------|
| 用户做出决策 | 决策内容 + 理由 |
| 解决了一个问题 | 问题描述 + 解决方案 |
| 用户说"记住这个" | 完整内容 |
| 重要配置变更 | 变更前后 + 原因 |
| session 出现错误 | 错误内容 + 教训 |

### Session 过大时的主动处理

**优先用 context 使用率**（通过 `session_status` 工具获取）：

| Context 使用率 | 状态 | 我的行动 |
|--------------|------|----------|
| < 60% | ✅ 健康 | 正常使用 |
| 60–80% | ⚠️ 注意 | 确保日记和任务状态已写入 |
| > 80% | 🔴 主动处理 | 写入日记和任务状态，告知用户建议 `/new` |

## Part 2：项目索引

### 存储位置

项目索引维护在 `MEMORY.md` 的 `## 项目索引` 章节中。

### 项目条目格式

```markdown
### {项目名称}
- **用途**：{一句话说明这个项目是什么}
- **本地目录**：`{绝对路径}`
- **Git 地址**：{https://github.com/... 或 "无（本地项目）"}
- **当前版本**：{版本号或状态描述}
- **最后更新**：{YYYY-MM-DD}
- **注意**：{关键注意事项，下次操作前必须知道的事}
```

### 必须更新项目索引的时机

| 触发 | 操作 |
|------|------|
| 新增项目/skill | 新建条目 |
| 发布新版本到 ClaWHub/GitHub | 更新版本号和最后更新日期 |
| 更改 Git remote 或本地目录 | 更新对应字段 |
| 删除项目/skill | 从索引中移除对应条目 |
| 发现重要注意事项 | 追加到"注意"字段 |

> ⚠️ **任何项目操作完成后，必须同步更新 MEMORY.md 项目索引。** 创建、删除、版本更新都算——操作完不更新索引等于没记录。

## Part 3：任务状态管理

### 存储位置

任务状态维护在 `memory/tasks.md`。

### tasks.md 格式

```markdown
# 任务状态

## 进行中
- [ ] {任务描述}
  - 开始：{YYYY-MM-DD}
  - 背景：{足够让下一个 session 理解上下文的关键信息}
  - 下一步：{具体的下一个操作步骤}

## 最近完成
- [x] {任务}（完成：{YYYY-MM-DD}）
```

### 必须更新 tasks.md 的时机

| 触发 | 操作 |
|------|------|
| 开始一个需要多轮对话的任务 | 新增"进行中"条目 |
| 任务完成 | 移到"最近完成"，标记日期 |
| 任务被搁置 | 在条目下补充"搁置原因"和"恢复条件" |
| 即将执行 `/new` | 确保所有进行中的任务已记录下一步 |

### 注意事项

- "下一步"要写得足够具体，让全新 session 的 AI 无需追问就能直接执行
- 已完成的任务保留最近 7 天，之后可归档到日记
- 日记不要记录密钥、token 等敏感信息
